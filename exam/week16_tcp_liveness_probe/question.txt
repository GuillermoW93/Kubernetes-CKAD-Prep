Create a pod in the session283884 namespace named tcp-check using image nginx with a TCP liveness probe on port 80.

apiVersion: v1
kind: Pod
metadata:
  name: tcp-check
  namespace: session283884
spec:
  containers:
  - name: tcp-check
    image: nginx
    livenessProbe:
      tcpSocket:
        port: 80
      initialDelaySeconds: 15
      periodSeconds: 10

With 'kubectl describe pod tcp-check -n session283884' you can verify the TCP Liveness Probe. If it fails at the bottom under 'Events' you should see:
- Warning  Unhealthy  10s  kubelet  Liveness probe failed: tcp probe failed
